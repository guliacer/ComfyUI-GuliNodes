# ComfyUI-GuliNodes

一个面向中文工作流习惯的 ComfyUI 实用节点合集，重点补足日常出图里最常用、最容易重复操作的几个环节：工具栏编排、节点上色、尺寸适配、图像处理、LoRA 叠加、文本整理、模型加载、图像反推与工作流辅助。

下面的推荐星级采用更严格的标准，综合考虑通用性、稳定性、实际效率提升、效果一致性与可替代性，满分 5 星。

### 1. GG Toolbar

这是本插件最值得优先体验的前端增强功能，也是最能直接提升 ComfyUI 画布操作效率的一部分。

功能作用：

- 为 ComfyUI 画布提供一个独立的悬浮工具栏
- 支持节点上色，并可区分“整个节点 / 节点内部 / 标题栏”三种上色模式
- 支持一组预设颜色与两组自定义颜色，适合统一工作流视觉分层
- 支持一键清除节点颜色
- 支持节点等宽、等高、左对齐、右对齐、顶部对齐、底部对齐、水平居中、垂直居中
- 支持自动间距整理，适合批量整理一排节点
- 支持紧凑化宽度，快速压缩节点占位
- 支持工具栏自身背景色与透明度调整，并会记住本地设置

基础使用方式：

1. 安装插件并重启 ComfyUI 后，工具栏默认显示在画布底部中央。
2. 工具栏支持拖动：按住工具栏空白区域 `【鼠标左键】` 拖拽，可以把它移动到你习惯的位置。
3. `【鼠标右键】` 点击工具栏可立即隐藏，隐藏后会在原位置显示一个迷你图标。
4. 点击迷你图标可重新显示完整工具栏。
5. `【鼠标右键】` 点击迷你图标也可以恢复完整工具栏。
6. `【鼠标中键】` 点击工具栏，可打开工具栏样式设置面板。
7. 在样式设置面板里可以调整工具栏背景颜色和透明度，设置会自动记住。

#### 上色工具栏

上色工具栏位于工具栏上半部分，主要负责节点、分组与工作流结构的视觉整理。

支持对象：

- 单个节点
- 多个已选中的节点
- 分组
- 多个已选中的节点与分组颜色整理

推荐使用顺序：

1. 先确定上色模式。
2. 再选择颜色。
3. 点击画笔按钮启用上色。
4. 最后点击节点，或提前选中多个节点后统一处理。

按钮详细说明：

1. `上色按钮`
- 作用：开启或关闭“点击节点即可上色”的画笔模式。
- 怎么用：先选颜色，再点击这个按钮启用上色，之后用 `【鼠标左键】` 点击节点即可上色。
- 使用细节：
- 已选好颜色并启用画笔后，点击任意节点，就会立即应用该颜色。
- 如果你已经选中了多个节点，再点击其中一个已选中节点，颜色会同时应用到这一批已选中的节点。
- 如果点击的是分组，分组也会被上色。
- 再次点击这个按钮，可以关闭上色模式，避免误操作。

2. `上色模式按钮`
- 作用：切换颜色要作用在节点的哪一部分。
- 怎么用：点击该按钮后，会弹出三个上色模式选项。
- 模式说明：
- `节点整体`：同时影响节点标题与主体区域，适合给整类功能统一颜色。
- `节点内部`：主要给节点主体区域上色，适合做轻量区分。
- `标题栏`：只给节点标题栏上色，适合保留节点主体默认样式时使用。

3. `预设颜色按钮`
- 作用：快速选择一组内置颜色。
- 怎么用：点击任意颜色圆点即可选中该颜色。
- 使用细节：
- 选中颜色后，如果画笔已经开启，下一次点击节点就会立刻上色。
- 如果再次点击当前已选中的同一个颜色，会取消当前颜色选择。

4. `自定义颜色 1 / 自定义颜色 2`
- 作用：保存你常用的两个自定义颜色。
- 怎么用：点击自定义颜色按钮，会弹出颜色选择器；选定后该颜色会保存下来。
- 使用细节：
- 新颜色会直接覆盖当前按钮颜色。
- 颜色会保存在本地，下次打开 ComfyUI 仍可继续使用。
- 选择自定义颜色后，也会成为当前上色颜色。

5. `删除颜色按钮`
- 作用：清除节点或分组的颜色。
- 怎么用：
- 如果你已经选中了一个或多个节点，直接点击此按钮，会立刻清除这些已选中节点的颜色。
- 如果没有选中节点，但画笔模式已开启，点击此按钮后会进入“点击节点删除颜色”的模式，接着再用 `【鼠标左键】` 点击节点即可删除颜色。
- 如果没有选中节点，也没有开启画笔，这个按钮不会直接处理，需要先选中节点或启用画笔。

上色工具栏的典型使用示例：

1. 想给某个节点立即上色：
- 先点击颜色，再点击上色按钮，再左键点击目标节点。

2. 想给多个节点统一上色：
- 先框选多个节点，再点击颜色，再点击上色按钮，最后点击这一组选中节点中的任意一个，即可一起上色。

3. 想给分组上色：
- 先选择颜色并启用上色，再点击分组区域即可。

4. 想清除多个节点颜色：
- 先选中多个节点，再点删除颜色按钮即可立即清除。

#### 尺寸调节工具栏

尺寸调节工具栏位于工具栏下半部分，主要负责节点排版、对齐与布局整理。

使用前提：

- 大部分按钮要求你先选中多个节点
- 少数按钮至少需要选中 1 个节点
- 如果数量不够，工具栏不会执行操作

按钮详细说明：

1. `自动宽度`
- 作用：把当前选中的多个节点统一成相同宽度。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会统一为这组节点里最大的宽度。
- 适合场景：让一组功能相近的节点看起来更整齐。

2. `自动高度`
- 作用：把当前选中的多个节点统一成相同高度。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会统一为这组节点里最大的高度。
- 适合场景：多行输入节点、文本节点或参数节点排版统一。

3. `最左对齐`
- 作用：让多个节点左边缘对齐。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会贴齐到最靠左那个节点的位置。

4. `最右对齐`
- 作用：让多个节点右边缘对齐。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会贴齐到最靠右那个节点的右边界。

5. `水平居中`
- 作用：让多个节点在垂直方向上排成同一水平线。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：节点会按中心高度对齐，适合横向排布一整行节点。

6. `最顶对齐`
- 作用：让多个节点顶部边缘对齐。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会贴齐到最上方节点的位置。

7. `最底对齐`
- 作用：让多个节点底部边缘对齐。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：所有选中节点会贴齐到最下方节点的底边。

8. `垂直居中`
- 作用：让多个节点在水平方向上排成同一竖列。
- 怎么用：至少选中 2 个节点后点击。
- 实际效果：节点会按中心宽度对齐，适合纵向堆叠布局。

9. `自动间距`
- 作用：自动整理多个节点之间的间距。
- 怎么用：至少选中 3 个节点后点击。
- 实际效果：
- 工具栏会根据前两个节点的相对位置，自动判断你当前更像是在做横向排列还是纵向排列。
- 然后使用前两个节点之间的距离作为基准，把后续节点按同样间距自动排开。
- 适合场景：快速整理一排采样链、文本链或预处理链。

10. `自适应尺寸（紧凑）`
- 作用：快速把选中节点的宽度压缩到较紧凑的统一状态。
- 怎么用：至少选中 1 个节点后点击。
- 实际效果：会把选中节点宽度统一调整为较紧凑的固定宽度，方便减少画布占用。
- 适合场景：节点过宽、画布拥挤、想压缩布局时使用。

工具栏样式设置说明：

`【鼠标中键】` 点击工具栏

- 作用：打开或关闭工具栏设置面板。
- 面板中可调内容：
- 工具栏背景颜色
- 工具栏透明度
- 一键恢复默认样式

`恢复默认样式按钮`

- 作用：把工具栏颜色和不透明度恢复成默认值。
- 适合场景：修改过多后想快速回到初始状态。

工具栏隐藏与恢复说明：

`【鼠标右键】` 点击完整工具栏

- 作用：隐藏完整工具栏。

`【鼠标左键】` 点击迷你图标

- 作用：重新显示完整工具栏。

`【鼠标右键】` 点击迷你图标

- 作用：同样可以恢复完整工具栏。

优势是什么：

- 比手动拖节点、一个个改颜色快很多
- 对大型工作流非常友好，能明显降低画布凌乱感
- 既解决“视觉整理”，也解决“批量排版”
- 本地记忆工具栏样式和节点颜色状态，长期使用更顺手

推荐星级：

`★★★★☆`

### 2. GG CLIP文本编码器 + GG 剪贴板读取

这两个节点建议配合使用，可以理解为一组“文本快速进入工作流”的高频辅助工具。

功能作用：

- `GG 剪贴板读取` 负责把系统剪贴板中的文本快速读入 ComfyUI
- `GG CLIP文本编码器` 负责把文本直接编码为 `CONDITIONING`
- 两者组合后，可以把外部 Prompt、标签、描述文本快速导入并立刻参与采样流程

怎么使用：

1. 先在外部文档、网页、提示词站或笔记工具中复制文本。
2. 使用 `GG 剪贴板读取` 读取内容，或直接在 `GG CLIP文本编码器` 中点击“读取剪贴板”按钮。
3. 如果只是想把文本送入后续文本节点，可从 `GG 剪贴板读取` 输出字符串继续处理。
4. 如果要直接参与生成，则把 `CLIP` 模型接入 `GG CLIP文本编码器`，输出 `CONDITIONING` 到采样链路。

优势是什么：

- 解决“外部文本导入 + 条件编码”这条链路里的重复操作
- 非常适合中文 Prompt 快速试验、批量提示词整理和临时灵感记录
- 比手动粘贴、再接编码节点更顺手，尤其适合高频调词场景

推荐星级：

`★★★★☆`

### 3. GG 图像压缩

这个节点服务于“出图后整理与交付”，压缩方式可以在节点内三选一。

包含节点：

- `GG 图像压缩`
- `GG 图像保存`

功能作用：

- 将 ComfyUI 输出结果压缩为更适合保存、分享、上传的图像
- 压缩节点只负责压缩并继续输出图像，不直接保存文件
- 保存格式由 `GG 图像保存` 控制，支持 JPEG、PNG、WEBP 和自动

怎么使用：

1. 将图像接入 `GG 图像压缩`。
2. 在 `压缩方式` 中选择 `civilblur`、`Caesium` 或 `meowtec`，默认是 `civilblur`。
3. 按需要调整 `质量` 和 `目标大小KB`。
4. 将压缩后的图像接入 `GG 图像保存`。
5. 在保存节点中选择 JPEG、PNG、WEBP 或自动格式。

优势是什么：

- 补上了 ComfyUI 工作流里常被忽略的“最终交付”环节
- 一个压缩节点集中管理三种方式，减少节点列表和参数混淆
- 对需要上传平台、发客户、做归档、批量整理结果图的用户特别实用

推荐选择建议：

- 追求画质优先：选择 `meowtec`
- 追求体积精控：选择 `Caesium`
- 追求轻量快速：选择 `civilblur`

推荐星级：

`★★★★☆`

## 视频节点使用提醒

本节用于说明视频节点的推荐连接方式，以及大文件加载时最常见的限制与处理建议。

### 适用节点

- `GG 视频加载`
- `GG 视频压缩`
- `GG 视频保存`

### 推荐使用方式

- 常规体积视频可优先使用 `GG 视频加载`。
- `GG 视频压缩` 仅负责压缩，不负责最终保存；压缩后的视频对象建议继续连接到 `GG 视频保存`。
- 如需切换输出格式，建议在 `GG 视频保存` 节点中统一选择目标格式。

### 关于 413 报错

- `413` 表示 `Content Too Large`，也就是浏览器上传到 ComfyUI 的单次请求体超过了服务端允许的大小。
- 这个限制通常发生在“加载视频文件”阶段，而不是发生在“视频压缩”阶段。
- 视频文件尤其是高码率、大分辨率素材，更容易触发这一限制。

### 大视频加载建议

- 如果需要压缩较大的视频文件，建议先将视频移动到 `ComfyUI/input` 目录，或先放到 `ComfyUI` 所在目录下，再进行加载。
- 这样可以减少浏览器上传大文件时触发 `413` 的概率，也能降低跨硬盘路径读取不稳定带来的影响。
- 对于大视频，优先使用本地工作目录内的视频文件进行加载，整体稳定性会更高。

### 预览与压缩说明

- `GG 视频加载` 与 `GG 视频保存` 已支持节点内视频预览，可直接播放并拖动进度条。
- `GG 视频压缩` 已增加实时进度显示，用于辅助判断当前压缩阶段是否正常推进。
- 如需读取较大视频，建议先移动到 `ComfyUI/input` 目录后再通过 `GG 视频加载` 选择。
- 如果遇到“能选择视频但无法预览”的情况，建议优先检查文件路径、格式，以及 ComfyUI 重启后的前端脚本是否已刷新。

更新内容：

- 新增 `GG 视频压缩` 节点，仅负责视频压缩，不直接保存文件，压缩后的视频对象可继续传递给后续节点。
- 新增 `GG 视频保存` 节点，负责将上游视频对象保存到输出目录，并支持切换 `mp4 / flv / mov / avi / f4v` 输出格式。

  `x264_32-8bit.exe`
  `x264_32-10bit.exe`
  `x264_64-8bit.exe`
  `x264_64-10bit.exe`
  `x264_64-12bit.exe`
  `x265-64-8bit.exe`
  `x265-64-10bit.exe`
  `x265-64-12bit.exe`
- 清理并去除了重复、冗余的编码器选项和多余输入输出项，连接关系更清晰。
- 优化视频节点之间的类型连接，`GG 视频压缩` 可更稳定地衔接 `GG 视频保存`。
- 优化本地视频加载逻辑，增强对输入目录视频文件的兼容性。
- 为 `GG 视频加载` 和 `GG 视频保存` 增加节点内视频预览，可直接播放并拖动进度条。
- 优化视频预览区域的节点内布局、自适应尺寸与节点高度联动，避免播放器超出节点范围。
- 为 `GG 视频压缩` 增加压缩过程的实时进度显示，尽量与 ComfyUI 原生执行进度保持一致。
- 新增视频节点的 `413` 原因说明。
- 新增大视频使用建议：如需压缩大文件，建议先将视频移动到 `ComfyUI/input` 或 `ComfyUI` 所在目录下，再进行加载。
- 补充说明：`413` 属于上传阶段限制，不属于视频压缩阶段报错。

适用价值：

- 让 ComfyUI 工作流内补齐“加载视频 -> 压缩视频 -> 保存视频”的完整链路。

- 通过节点内预览与实时进度反馈，提升视频节点的可视化体验与排查效率。
- 帮助用户区分“上传失败”和“压缩失败”。

- 提升视频节点在本地工作流中的可用性。
